00 — Product Soul · v2.0

产品灵魂 Cellular Intelligence OS

产品宣言 · Manifesto
大多数健康App告诉你你做了什么。
Coach.AI告诉你你的身体正在发生什么。

体重增加、糟糕的睡眠、无法摆脱的疼痛、慢性疲劳、高胆固醇—— 这些不是孤立的症状，而是细胞层面失衡的信号。 我们的使命是帮助用户看见这些信号之间的联系， 并通过营养变化、生活方式调整和整体细胞支持， 逐步引导身体回到平衡。

产品关注的三个维度

🥗
营养变化
不只是卡路里计数。关注微量营养素、抗氧化摄入、蛋白质完整性—— 这些是细胞膜、酶和激素合成的原材料。
镁 · 锌 · Omega-3 · 维生素D · 抗炎食物密度
🌙
生活方式变化
睡眠是细胞修复的主要窗口。昼夜节律、压力激素、运动刺激—— 这些决定细胞是在「修复模式」还是「应激模式」。
HRV · 深睡眠 · 昼夜节律一致性 · 运动类型 · 压力负荷
⚡
整体细胞支持
追踪那些顽固问题的根源——慢性炎症、线粒体效率、代谢稳定性—— 让难以解决的问题开始得到关注。
疼痛信号 · 能量质量 · 体重趋势 · 代谢灵活性 · 激素平衡

✦ 翻译者——把数据翻译成身体感受的语言

09 — Guided Discovery

引导式发现原则 Anti-Dashboard Philosophy

用户不需要看懂数据。
用户需要被引导着感受他们的身体。
G1
每天一个真相
不展示20个指标，而是由AI决定今天最重要的一个洞察， 用一句话说清楚。其他数据在用户主动探索时才显现。

"你的细胞今天在修复模式。这是三天累积的结果。"
G2
叙事先于数字
先说「发生了什么」，再说「数字是多少」。 用户只有在理解了叙事之后，数字才有意义。

"你的睡眠修复质量不够" 先于 "深睡眠17%"
G3
因果链条可视化
连接用户自己看不见的点。上周的行为导致了今天的状态。 让用户体验到「原来是这样」的顿悟，而不是孤立地看一个数字。

周二深夜工作 → 周三浅睡眠 → 周四HRV低 → 今天疲惫
G4
渐进式揭示
信息分3层：感受层（直觉）→ 理解层（原因）→ 行动层（建议）。 用户主动点击才能看到更深一层，不强迫，不淹没。

触摸状态环 → 展开四项评分 → 再触摸查看每项建议
G5
身体语言，非临床语言
用身体感受描述细胞状态，不用医学术语。 让用户感到被理解，而不是被检测。

"充电中" 不是 "副交感神经激活"
G6
温暖而非恐吓
红色状态不是警报，是关心的提醒。 所有建议都以「支持」的语气表达，不用「你应该/你必须」。

"今天适合轻柔地支持你的身体" 不是 "警告：恢复不足"

10 — Cellular Health Index

细胞健康评分 CHI · Cellular Health Index

CHI 是产品的核心智能——一个将 HRV、睡眠质量、营养状态和生活方式四项维度 综合成单一可感知评分的算法。它不是一个数字仪表盘，而是一个 身体故事的起点。

四个支柱 · 权重分配

P1 · 恢复状态
Recovery State
35%
反映自主神经系统状态——身体是否从上一个周期完全恢复。 是细胞「能量预算」的核心指标。
输入信号
HRV今日值 · HRV周趋势斜率 · 静息心率 · HRV相对基线偏差
P2 · 细胞修复质量
Cellular Repair Quality
25%
深睡眠是细胞自噬（autophagy）的主要窗口，REM是神经修复期。 这两者的质量直接决定细胞的「夜间维护」效果。
输入信号
深睡眠百分比 · REM百分比 · 睡眠总时长 · 觉醒次数 · 睡眠一致性
P3 · 营养支持
Nutritional Support
25%
细胞的建材和燃料来源。蛋白质充足度决定修复能力， 卡路里范围决定代谢稳定性，碳水/脂肪平衡影响炎症倾向。
输入信号
蛋白质达标率 · 卡路里在范围内 · 碳水控制 · 脂肪质量 · 记录完整度
P4 · 生活方式一致性
Lifestyle Coherence
15%
细胞最喜欢节律和预期。昼夜节律一致、运动刺激规律、 压力负荷可控——这些「环境因素」决定细胞是否感到安全。
输入信号
睡眠时间一致性 · 本周运动次数 · 运动多样性 · 连续记录天数

评分算法逻辑

// ── 各支柱单项评分（0–100）────────────────────────── // P1 恢复状态 · 35% P1 = normalize(hrv, baseline) // HRV偏离个人基线的标准化值 + bonus(hrv_trend_7d > 0, +8) // 周趋势上升加分 + penalty(rhr > 59, -12) // 静息心率过高扣分 // P2 细胞修复质量 · 25% P2 = score(deep_pct, target=20) // 深睡比例，目标20% + score(rem_pct, target=22) // REM比例，目标22% + score(sleep_h, target=7.5) // 睡眠时长，目标7.5h + penalty(awake >= 4, -10) // 频繁觉醒扣分 // P3 营养支持 · 25% P3 = score(protein / target, threshold=0.7) // 蛋白质达标率，70%起步 + score(cal_in_range(0.5, 1.1)) // 卡路里在目标50–110%内 + score(carbs <= carb_target) // 碳水不超标 + score(fat <= fat_target) // 脂肪不超标 × completeness_factor(meals_logged) // 记录完整度系数 0.5–1.0 // P4 生活方式一致性 · 15% P4 = score(sleep_time_consistency_7d) // 近7天入睡时间标准差 + score(workout_days_7d, target=3) // 近7天运动天数 + bonus(streak >= 3, +6) // 连续记录3天以上加分 // ── 综合 CHI ──────────────────────────────────────── CHI = 0.35 × P1 + 0.25 × P2 + 0.25 × P3 + 0.15 × P4 // ── 状态映射 ───────────────────────────────────────── state = CHI >= 75 ? "细胞活跃 · Thriving" : CHI >= 55 ? "细胞维稳 · Maintaining" : CHI >= 40 ? "细胞应激 · Stressed" : "细胞耗竭 · Depleted"

01 — Design Philosophy

设计哲学

01
情绪即材质
色彩不只是装饰，它是身体状态的直接映射。绿色 = 你今天很好，红色 = 需要关注，不需要用户思考。
02
数字优先
核心数据用最大的字号、最轻的字重呈现。数字即信息，文字是辅助说明，从不喧宾夺主。
03
空间即奢侈
大留白是产品高级感的核心。不压缩、不塞满，让每一个数据都有呼吸的空间。
04
70 · 20 · 10 法则
70% 中性底色（Mineral Graphite 系列）承载所有内容
20% 情感辅色（Dust Rose / Aerobic Blue）增加温度
10% 能量强调（Recovery Aqua / Butter Energy）传递状态
05
有机 · 非几何
动效模拟呼吸与心跳节律，而非机械弹跳。
圆角使用有机渐变，避免等比正圆。
颜色用暖调而非纯冷色，传达「身体感知」而非「仪表盘」。

06 — Motion System

动效原则 Motion System

所有动效模拟身体节律，而非机械弹跳。速度慢 = 平静恢复，速度快 = 运动能量。

Breathe · 呼吸
用于状态环 · HRV 脉冲。模拟深呼吸节律，传达恢复感。
scale: 0.88 → 1.12
duration: 4s · ease-in-out · infinite
🏃
Calm Float · 悬浮
用于图标轻微浮动 · 卡片加载动效。像水面浮动的轻盈感。
translateY: 0 → -4px
duration: 3s · ease-in-out · infinite
⚡
Snap · 即时响应
用于按钮点击 · Tab切换 · 数据加载完成。传达精准、有力的控制感。
duration: 120–180ms
easing: cubic-bezier(0.4, 0, 0.2, 1)
💤
Sleep Fade · 睡眠渐显
用于睡眠页面切换 · 夜间数据展示。极慢淡入，模拟入睡状态。
opacity: 0 → 1
duration: 800ms · ease-out
📊
Chart Draw · 图表绘制
用于 HRV 周图 · 营养进度条。从左到右顺序绘制，让用户跟随数据流动。
stagger: 60ms per item
duration: 300ms · ease-out
📱

duration: 320ms · cubic-bezier(0.16,1,0.3,1)

08 — Usage Rules

使用规范 Do & Don't

✓ DO · 应当
核心数字（HRV/心率/睡眠）用 #FFFFFF 纯白
标题文字用 #F0EBE4 Title Warm
状态色背景透明度用 20%（不是12%）
状态环加对应语义色发光（blur 20–24px）
卡片用渐变底色 #252B32→#2B3138
用 Recovery Aqua 表示健康/恢复状态
保持大量留白，不要填满屏幕
颜色遵守 70/20/10 比例
Butter Energy 仅作点缀（<10%面积）
✕ DON'T · 不应
用语义色（#59C3C3）直接作为数字颜色
在普通卡片上使用发光效果
卡片用平色背景（缺乏深度感）
用纯黑 #000 作为任何背景
混用两种语义色于同一张卡片
用 Navy Blue / 高饱和蓝作主色
使用 bounce/spring 弹跳动效
Recovery Aqua 用于非健康状态场景
Dust Rose 作为警示/错误状态颜色
